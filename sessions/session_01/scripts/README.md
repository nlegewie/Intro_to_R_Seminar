# Skripte zu Session 1

Für die **Endgame-Demo** (`demo_endgame.Rmd`) liegt der gesamte Analysecode **direkt im R Markdown**, damit beim Aufklappen von *Code* der echte `dplyr`/`ggplot2`-Code sichtbar ist — nicht nur `source()`.

Später im Seminar und im Abschlussbericht wirst du dieselbe Logik typischerweise in **`.R`-Skripte** auslagern und mit `source(here::here("scripts", "..."))` einbinden. Die Demo zeigt bewusst alles an einem Ort; der Kurs zeigt dir dann die saubere Projektstruktur.
